2. Logistic Regression

Despite the name, logistic regression is a classification algorithm. It applies the sigmoid function to a linear combination to output a probability between 0 and 1.

P(y=1|x) = σ(Wx + b) = 1 / (1 + e⁻⁽ᵂˣ⁺ᵇ⁾)

3. Decision Trees

A decision tree splits data by asking yes/no questions about features, choosing the split that best separates the classes (or reduces variance for regression).

Trees: Pros & Cons
  • Pros: Interpretable, handles mixed data types, no scaling needed, captures non-linear patterns
  • Cons: Overfits easily (high variance), unstable (small data changes → different tree), greedy splits aren't globally optimal
  • Fix: Pruning (max_depth, min_samples_leaf) or use ensembles (Random Forest, XGBoost)

4. Ensemble Methods

Combine multiple weak models to create a strong one. The two main strategies: bagging (parallel, reduce variance) and boosting (sequential, reduce bias).

5. Unsupervised Learning: K-Means Clustering

No labels — the algorithm discovers structure on its own. K-Means partitions data into K clusters by iteratively assigning points to the nearest centroid and updating centroids.

6. Bias-Variance Tradeoff & Regularization

Total Error = Bias² + Variance + Irreducible Noise
The Practical Rule

If training accuracy is high but test accuracy is low → overfitting → add regularization, dropout, or get more data. If both are low → underfitting → use a more complex model or better features. Cross-validation is your best tool for detecting this.
